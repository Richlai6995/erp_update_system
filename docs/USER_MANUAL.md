# ERP 程式更新系統 - 使用者操作手冊 (User Manual)

歡迎使用「ERP 程式更新系統」。本系統提供 ERP 程式 (Form, Report, SQL, Library) 的版本更新申請、簽核與佈署流程。

## 1. 登入系統

*   請使用您的 **AD 網域帳號 (Windows 登入帳號)** 與密碼進行登入。
*   若您是外部人員或無 AD 帳號，請聯繫系統管理員索取本機帳號。
*   預設密碼通常由管理員提供，首次登入後無法自行修改本機密碼 (需聯繫管理員)。

## 2. 系統主畫面 (Dashboard)

登入後，您將看到申請單列表：
*   **查詢功能**：點擊「查詢條件」展開篩選面板，可依單號、日期、申請人、部門或程式類別進行搜尋。
*   **狀態篩選**：上方標籤可快速切換查看「簽核中」、「已結案」或「被退回」的申請單。
*   **ERP 檔案查詢**：點擊「ERP檔案查詢與下載」分頁，可瀏覽伺服器上現有的程式版本 (僅限查詢)。

## 3. 建立更新申請單

1.  在主畫面點擊右上角的 **「+ 建立申請單」** 按鈕。
2.  填寫基本資訊：
    *   **程式類別**：選擇 Form (畫面), Report (報表), SQL, Library 或 DB Object。
    *   **ERP 模組**：選擇所屬模組 (如 GL, AP, INV 等)。
    *   **需求描述**：請清楚說明此次更新的目的或修改內容。
3.  點擊「建立」後，將進入詳細編輯頁面。

## 4. 檔案上傳

在申請單詳細頁面中：
1.  點擊 **「上傳檔案」** 區域，或直接將檔案拖曳至該區塊。
2.  **支援格式**：`.fmb`, `.fmx`, `.rdf`, `.jsp`, `.sql`, `.pll`, `.plx` 等。
3.  **檔案說明**：上傳後，您可以對個別檔案輸入備註說明。
4.  **刪除檔案**：在尚未提交前，您可以刪除已上傳的錯誤檔案。

## 5. 提交簽核

1.  確認所有資訊填寫正確，且檔案皆已上傳。
2.  點擊下方的 **「送出簽核 (Submit)」** 按鈕。
3.  送出後，狀態將變更為 **「簽核中 (Reviewing)」**，系統將自動寄信通知您的部門主管。
4.  **注意**：送出後即無法修改內容。如需修改，需請主管退回。

## 6. 狀態追蹤與通知

*   **簽核流程**：
    *   **主管簽核**：由申請人部門主管審核。
    *   **DBA 佈署**：主管同意後，由 DBA 進行程式上線與佈署。
*   **Email 通知**：
    *   當您送出申請時，主管會收到通知。
    *   當主管或 DBA 同意/退回時，您會收到結果通知。
    *   當程式正式上線 (已上線 - Online) 時，您會收到結案通知。

## 7. 注意事項

*   **檔案命名**：請勿使用中文檔名或特殊符號，建議依照 ERP 標準命名規範。
*   **重複檔名**：若系統偵測到重複檔名，可能會阻擋上傳，請確認是否為同一檔案。
*   **機密資訊**：請勿上傳包含資料庫密碼、金鑰或其他機密設定的檔案 (如 `.env`, `config.js`)。
*   **作廢申請**：若申請單已不再需要，可由申請人或管理員執行「作廢」 (Void)，狀態變為 **「已作廢 (Void)」**，作廢後無法復原。
